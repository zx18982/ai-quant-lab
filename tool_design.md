## 一、产品概述

### 1.1 产品定位

一款纯前端的**股票技术指标交互式可视化工具**，用户选择股票后即可实时计算并展示 RSI、MACD、布林带、ATR 四个技术指标，所有指标参数均可自由调节，图表即时重绘。

### 1.2 核心价值

| 痛点 | 解决方案 |
|------|---------|
| 写 Notebook 看指标，改参数要重跑全流程 | 参数滑块拖动即重绘，秒级响应 |
| 不同股票要分别写代码 | 下拉切换股票，自动加载数据 |
| 参数调优看不到对比效果 | 实时反馈，所见即所得 |
| 指标含义不直观 | 图表标注信号区域，参数旁附说明 |

### 1.3 目标用户

- 金融/量化学习者：理解技术指标的计算逻辑和参数敏感性
投资分析师：快速对比多只股票的技术指标状态
- 在线实习项目成员：作为 task02 的交互式成果展示

### 1.4 使用场景

```
用户打开页面 → 选择股票 → 查看默认参数下的四指标图表
→ 拖动参数滑块 → 图表实时重绘 → 观察参数变化对指标的影响
→ 切换股票对比 → 导出截图或数据
```

| **布林带** | 可调参数：周期、标准差倍数 | P0 |
| **ATR 指标** | 可调参数：周期、止损倍数 | P0 |

### 2.2 参数体系

每个指标的参数定义，包括默认值、范围、步长：

#### RSI — 相对强弱指标

| 参数 | 键名 | 默认值 | 范围 | 步长 | 说明 |
|------|------|--------|------|------|------|
| 周期 | `period` | 14 | 2~50 | 1 | 计算RSI的天数 |
| 超买线 | `overbought` | 70 | 50~90 | 1 | 超买阈值 |
| 超卖线 | `oversold` | 30 | 10~50 | 1 | 超卖阈值 |

约束：`overbought > oversold`

#### MACD — 指数移动平均收敛/发散

| 参数 | 键名 | 默认值 | 范围 | 步长 | 说明 |
|------|------|--------|------|------|------|
| 快线周期 | `fast_period` | 12 | 2~50 | 1 | EMA快线天数 |
| 慢线周期 | `slow_period` | 26 | 5~100 | 1 | EMA慢线天数 |
| 信号线周期 | `signal_period` | 9 | 2~50 | 1 | DEA信号线天数 |

约束：`slow_period > fast_period`

#### 布林带 — Bollinger Bands

| 参数 | 键名 | 默认值 | 范围 | 步长 | 说明 |
|------|------|--------|------|------|------|
| 周期 | `period` | 20 | 5~100 | 1 | 中轨SMA天数 |
| 标准差倍数 | `std_dev` | 2.0 | 0.5~4.0 | 0.1 | 上下轨的标准差倍数 |

#### ATR — 平均真实波幅

| 参数 | 键名 | 默认值 | 范围 | 步长 | 说明 |
|------|------|--------|------|------|------|
| 周期 | `period` | 14 | 2~50 | 1 | ATR计算天数 |
| 止损倍数 | `stop_multiplier` | 2.0 | 0.5~5.0 | 0.1 | 动态止损 = 价格 ± N×ATR |

## 四、交互设计

### 4.1 核心交互流

```
用户选择股票
    ↓
工具自动获取数据（显示加载状态）
    ↓
数据就绪 → 计算四指标（默认参数）→ 渲染所有图表
    ↓
用户拖动参数滑块
    ↓
防抖 150ms → 重算该指标 → 重绘对应子图（其他子图不重绘）
    ↓
用户继续调参 / 切换股票 / 导出数据
```

### 4.2 参数调节交互细节

**滑块拖动**：
- 拖动过程中实时显示当前值（tooltip 跟随光标）
- 释放后触发重绘（防抖 150ms，避免频繁计算）
- 支持点击数值直接键盘输入

**参数约束校验**：
- MACD：若 `slow_period ≤ fast_period`，滑块标红 + 提示"慢线周期须大于快线周期"，禁止重绘
- RSI：若 `overbought ≤ oversold`，同样标红提示

**开关切换**：
- 关闭某指标 → 对应子图淡出隐藏，其他子图高度自适应重新分配
- 开启某指标 → 子图淡入显示

### 4.3 加载状态

| 场景 | 表现 |
|------|------|
| 数据加载中 | 图表区显示骨架屏 + 加载动画 |
| 数据加载失败 | 显示错误提示 + 重试按钮 |
| 指标计算中 | 对应子图显示半透明遮罩 + 计算中提示 |
| 无数据 | 显示"该股票暂无数据"空状态 |

### 4.4 信号标注

在图表上自动标注关键信号点：

| 信号 | 标注方式 |
|------|---------|
| RSI 超买/超卖 | 对应区域背景填充（浅红/浅绿） |
| MACD 金叉 | 图表上标记绿色向上箭头 ↑ |
| MACD 死叉 | 图表上标记红色向下箭头 ↓ |
| 布林带触及上/下轨 | 触点处标注小圆点 |
| 布林带收窄 | 带宽低位区域背景变浅 |

底栏信号摘要实时更新，如：
`RSI 72.3 (超买) · MACD 金叉 (2天前) · 布林带带宽 8.2% (正常) · ATR 0.85 (偏高)`

## 五、技术架构

### 5.1 技术选型

| 层 | 技术 | 理由 |
|----|------|------|
| 前端框架 | 原生 HTML + CSS + JavaScript | 纯前端，无需构建工具，可直接用浏览器打开 |
| 图表引擎 | Chart.js 或 ECharts | ECharts 更适合金融图表（K线、十字光标、缩放） |
| 数据计算 | 原生 JS 实现 | 指标计算逻辑简单，无需引入计算库 |
| 数据来源 | 预置 JSON + 可选 API | 离线优先，支持在线获取 |

### 5.2 数据流

### 5.5 性能考量

| 关注点 | 策略 |
|--------|------|
| 参数拖动频繁触发 | 防抖 150ms，拖动结束后才重算 |
| 增量重绘 | 只重绘参数变化的指标子图，其他子图不动 |
| 数据缓存 | 切换回已加载过的股票时直接使用缓存 |
| 计算量 | 250条数据 × 4指标，计算量极小，无需 Web Worker |
| 图表渲染 | 使用 ECharts 的 `setOption` 增量更新，而非销毁重建 |

## 八、验收标准

1. 浏览器直接打开 `index.html` 即可使用，无需服务器
2. 可切换至少 3 只股票，切换后图表自动更新
3. 四个指标均可独立开关，关闭后对应子图隐藏
4. 每个指标的参数滑块可拖动调节，释放后图表在 200ms 内重绘
5. 参数约束校验有效（MACD 慢线 > 快线，RSI 超买 > 超卖）
6. 图表悬停显示十字光标和数值 tooltip
7. 底栏信号摘要随参数变化实时更新
8. 配色遵循涨红跌绿惯例
9. 响应式：在 768px 以上正常显示完整布局
10. 恢复默认参数按钮可一键重置所有指标参数
